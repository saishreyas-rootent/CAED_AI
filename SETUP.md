# CAD AI – React + FastAPI Integration Guide

> Replace Streamlit with a production-grade React UI backed by FastAPI.  
> Your existing `src/converter.py` and `src/raster_to_dxf.py` are **untouched**.

## Step 1 – Install Python dependencies

Add these three packages to your existing environment:

```bash
pip install fastapi uvicorn[standard] python-multipart
```

Or install everything fresh from the updated requirements file:

```bash
# rename the file first
mv requirements_new.txt requirements.txt

pip install -r requirements.txt
```

> `python-multipart` is **required** for FastAPI to accept file uploads.  
> `uvicorn` is the ASGI server that runs FastAPI.

---

## Step 2 – Copy the new files into your project

Place every file exactly where shown in the structure above.

```
api.py          → pdf_to_dxf/api.py
run.py          → pdf_to_dxf/run.py
frontend/       → pdf_to_dxf/frontend/   (entire folder)
```

---

## Step 3 – Install Node dependencies

```bash
cd frontend
npm install
cd ..
```

This reads `frontend/package.json` and installs React + Vite locally into `frontend/node_modules/`.  
It only needs to be run **once** (or after adding new npm packages).

---

## Step 4 – Run the app

### Option A – Single command (recommended)

```bash
python run.py
```

This starts both servers simultaneously and prints their URLs:

```
──────────────────────────────────────────────────────────
  API  → http://localhost:8000
  UI   → http://localhost:5173
  Docs → http://localhost:8000/docs
──────────────────────────────────────────────────────────
Press Ctrl+C to stop both servers.
```

Open **http://localhost:5173** in your browser.

### Option B – Two separate terminals

**Terminal 1 – FastAPI backend:**

```bash
uvicorn api:app --reload --port 8000
```

**Terminal 2 – Vite frontend:**

```bash
cd frontend
npm run dev
```

Open **http://localhost:5173** in your browser.

> `--reload` makes FastAPI restart automatically when you edit Python files.  
> Vite's dev server does the same for React files automatically.

---

## How the two servers connect

During development, Vite proxies all `/api/*` requests to FastAPI:

```
Browser → http://localhost:5173/api/pdf/convert
        ↓  (Vite proxy, transparent)
FastAPI → http://localhost:8000/pdf/convert
```

This is configured in `frontend/vite.config.js` — you do not need to change anything.

---

## API endpoints reference

| Method | Endpoint | What it does |
|--------|----------|-------------|
| GET | `/health` | Health check |
| POST | `/pdf/info` | Returns page count for an uploaded PDF |
| POST | `/pdf/preview` | Returns a PNG preview of one PDF page |
| POST | `/pdf/convert` | Converts PDF(s) → DXF or ZIP |
| POST | `/raster/preview` | Returns binarized PNG preview |
| POST | `/raster/convert` | Converts raster image → DXF |

Interactive API docs (auto-generated by FastAPI):  
**http://localhost:8000/docs**

## Building for production (optional)

When you are ready to deploy, build the React app into static files that FastAPI can serve directly:

```bash
cd frontend
npm run build
```

This outputs to `pdf_to_dxf/static/` (configured in `vite.config.js`).

Then add static file serving to `api.py`:

```python
from fastapi.staticfiles import StaticFiles

# Add this near the bottom of api.py, after all route definitions
app.mount("/", StaticFiles(directory="static", html=True), name="static")
```

Run only the FastAPI server:

```bash
uvicorn api:app --host 0.0.0.0 --port 8000
```

The full app (UI + API) is now served from a single port: **http://localhost:8000**

## Replacing the logo

The current logo in `App.jsx` is a placeholder SVG.  
To use your real rootent logo:

1. Copy your logo file to `frontend/src/assets/logo.png` (or `.svg`)
2. Open `frontend/src/App.jsx`
3. Replace the `LOGO_SRC` constant at the top:

```jsx
// Remove this line:
const LOGO_SRC = "data:image/svg+xml,..."

// Add this import at the top of the file:
import logo from './assets/logo.png'

// Then use it in the JSX (it's already referenced as LOGO_SRC):
const LOGO_SRC = logo
```

## Troubleshooting

### `ModuleNotFoundError: No module named 'fastapi'`
```bash
pip install fastapi uvicorn[standard] python-multipart
```

### `ModuleNotFoundError: No module named 'multipart'`
```bash
pip install python-multipart
```

### `sh: vite: command not found` or `'vite' is not recognized`
```bash
cd frontend && npm install
```

### Port already in use
Change the port in the command:
```bash
uvicorn api:app --reload --port 8001
```
Then update `frontend/vite.config.js`, replacing `8000` with `8001` in the proxy target.

### CORS errors in the browser
The `api.py` already includes `allow_origins=["*"]` which permits all origins during development. No changes needed.

### `npm run dev` works but the API calls fail
Make sure FastAPI is running (`python run.py` or `uvicorn api:app --reload --port 8000`).  
Check http://localhost:8000/health — it should return `{"status":"ok"}`.
